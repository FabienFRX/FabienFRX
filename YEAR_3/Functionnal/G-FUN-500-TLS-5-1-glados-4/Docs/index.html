Important Notes

  • Part 0 (Testing) is mandatory
    Provide unit, integration tests and CI. Tests are expected at every defense.
  • Behave like Chez-Scheme
    Match grammar, evaluation rules and observable behavior for the required features.
  • Errors to stderr with exit code 84
    Parsing/evaluation errors must not print to stdout and must exit with code 84.
  • Support file input and interactive REPL
    Accept program files (argv/stdin) and provide an interactive mode with -i.
  • Part 2 improvements (4 axes)
    Security, custom syntax, evaluation (VM/bytecode), and documentation enhancements.
