# My_Marvin - Jenkins CI/CD Automation

## 📌 Description
This repository contains the automated Jenkins configuration for the *My_Marvin* project, leveraging *Configuration as Code (JCasC)* and *Job DSL* for CI/CD pipeline management and automation.

### 🔥 Project Objectives
- Configure Jenkins using a YAML file (JCasC).
- Create users and manage roles with specific permissions.
- Use Job DSL to automate Jenkins job creation.
- Test and run Jenkins in a Docker environment.

## 🚀 Setup and Installation

### 1️⃣ Clone the Repository
```bash
git clone https://github.com/EpitechPromo2028/B-DOP-400-TLS-4-1-mymarvin-fabien.fraixanet.git
cd B-DOP-400-TLS-4-1-mymarvin-fabien.fraixanet
```

### 2️⃣ Set Up Environment Variables
Create a `.env` file:
```ini
USER_CHOCOLATEEN_PASSWORD=(set password)
USER_VAUGIE_G_PASSWORD=(set password)
USER_I_DONT_KNOW_PASSWORD=(set password)
USER_NASSO_PASSWORD=(set password)
```

### 3️⃣ Start Jenkins with Docker
```bash
docker-compose up -d
```
Jenkins will be accessible at **http://localhost:8080**.

### 4️⃣ Run the SEED Job
1. Log in to Jenkins.
2. Navigate to the **Tools** folder.
3. Run the **SEED** job, entering:
   - `GITHUB_NAME`: GitHub repository name (`owner/repo_name`)
   - `DISPLAY_NAME`: Display name for the generated job

4. Verify that the new job has been created and is working correctly!


## ⚙️ Features
### 🎯 **Jenkins Configuration as Code (JCasC)**
- Automatically configures Jenkins via `my_marvin.yml`.
- Disables user sign-up.
- Creates 4 users with passwords managed through environment variables.
- Sets up a role-based authorization strategy.

### 🛠 **Job DSL (Groovy)**
- **`clone-repository`**: Clones a specified Git repository.
- **`SEED`**: Automatically generates new jobs based on a GitHub repository.
- **Jobs generated by SEED**: Run `make` for compilation and automated tests.


## ❓ Troubleshooting
### 🔴 Jenkins won’t start?
✅ Check logs with:
```bash
docker-compose logs -f
```

### 🔴 The SEED job isn’t creating new jobs?
✅ Ensure the specified GitHub repository exists and is either public or accessible with credentials.

### 🔴 Incorrect user permissions?
✅ Verify the configuration in `my_marvin.yml` and restart Jenkins:
```bash
docker-compose down && docker-compose up -d
```
